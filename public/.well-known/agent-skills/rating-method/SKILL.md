# rating-method

## Когда использовать

- Когда нужно понять, что значит поле `rating`.
- Когда нужно объяснить сортировку по условному уровню.
- Когда нужно отделить рейтинг качества среды от цены.

## Главный URL

- Публичное объяснение методики: `/rating/`.

## Что важно помнить

- Рейтинг считается на этапе сборки сайта.
- Он зависит от текущей схемы данных и подтвержденных полей по поселку.
- Это технический proxy качества среды, а не рыночная оценка недвижимости.

## Что не стоит путать

- Тариф и итоговый рейтинг не одно и то же.
- Тариф намеренно исключен из формулы рейтинга.
- Низкая заполненность данных не должна автоматически трактоваться как плохое качество.

## Практический паттерн

- Для быстрой сортировки используй `rating` из `/data/explorer.json`.
- Для объяснения логики и ограничений ссылайся на `/rating/`.
