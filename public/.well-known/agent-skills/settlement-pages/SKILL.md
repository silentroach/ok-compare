# settlement-pages

## Когда использовать

- Когда уже известен `slug` поселка и нужен канонический HTML- или Markdown-URL по одному объекту.
- Когда нужно сослаться на страницу поселка или прочитать данные в человекочитаемом виде.
- Когда structured-данные уже взяты из `/data/settlements.json`, но нужна сама страница.

## Как строить URL

- Паттерн detail-страницы: `/settlements/[slug]/`.
- Обычно `slug` удобнее брать из `/data/settlements.json`.

## Что обычно есть на detail-странице

- каноническая HTML-страница и Markdown companion `/settlements/[slug]/index.md`
- те же подтвержденные факты, разложенные по секциям для чтения
- ссылки на сайт, Telegram и карту, пояснения по тарифу и адресный текст
- список `sources`, которого нет в общем full feed

## Ограничения

- Detail-страница удобна для чтения, но для массового анализа эффективнее полный feed.
- Если факт не подтвержден источником, он может отсутствовать.
- Отсутствие поля чаще значит «неизвестно», а не «точно нет».

## Практический паттерн

- Сначала используй `/data/settlements.json` для structured discovery.
- Затем переходи на `/settlements/[slug]/` для канонической страницы и точечной проверки одного поселка.
