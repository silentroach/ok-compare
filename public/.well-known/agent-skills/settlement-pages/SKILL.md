# settlement-pages

## Когда использовать

- Когда уже известен `slug` поселка и нужен полный контекст по одному объекту.
- Когда нужно проверить источники, сайт, Telegram, карту, детальную инфраструктуру и сервисную модель.
- Когда агрегированного `data/explorer.json` уже недостаточно.

## Как строить URL

- Паттерн detail-страницы: `/settlements/[slug]/`.
- Обычно `slug` удобнее брать из `/data/explorer.json`.

## Что обычно есть на detail-странице

- ссылки на сайт, Telegram и карту
- пояснения по тарифу и адресный текст
- `infrastructure`, `common_spaces`, `service_model`
- список `sources`

## Ограничения

- Detail-страница богаче общего feed, но все равно не должна трактоваться как гарантия полноты.
- Если факт не подтвержден источником, он может отсутствовать.
- Отсутствие поля чаще значит «неизвестно», а не «точно нет».

## Практический паттерн

- Сначала используй `/data/explorer.json` для discovery.
- Затем переходи на `/settlements/[slug]/` для точечной проверки одного поселка.
