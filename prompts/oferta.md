# Обновление оферт с сайта ОК Комфорт

Открой страницу https://ok-komfort.ru/objects/shelkovo-eco-club/ с помощью agent-browser, на ней расположены pdf-документы. Нас интересуют публичные договора.

Для каждого файла с публичным договором запустить субагента (не параллельно, а последовательно), который выполнит следующую последовательность действий с документом, учитывая его название из ссылки на странице:

- скачает его во временную папку
- запустит `ocrmypdf -l rus+eng --output-type pdfa --rotate-pages --deskew -O3 --jpeg-quality 85` для него, чтобы получить OCR-версию документа и сохранит в отдельный файл
- запустит `docker run --rm -i adeuxy/markitdown:latest < original.pdf > parsed.md` для OCR-версии документа, чтобы получить из него markdown содержимое
- пройдется по получившемуся markdown файлу и поправит опечатки и артефакты OCR-распознавания
- закинет получившиеся данные по договору в папку `apps/www/src/data/kb/ok/offer` с названием части поселка (смотри как называется ссылка на странице) в отдельную статью, где обязательно сверху должна быть специальная сноска, что документ распознан автоматически и стоит свериться с оригиналом (со слова сделай ссылку на страницу поселка, где есть все документы), после сноски должна быть отдельная строка под содержаний (`[TOC]` отдельной строкой)

До окончания выполнения задачи целиком нужно сохранять в отдельный файл процесс выполнения задачи, чтобы если что-то прервется - я перезапустил эту задачу и она продолжилась с того документа, который еще не был обработан.

После выполнения обработки всех документов нужно в индексном kb-документе для `kb/ok` сгруппировать все оферты в одну группу и списком дать ссылки на эти документы. Если одна часть поселка дробится на несколько - сгруппировать еще и в одну строчку.

## Нюансы

- Заголовок страницы должен быть "Публичная оферта - [Название части поселка с нюансами из названия со страницы на сайте]"
- Нельзя ничего придумывать и додумывать, только небольшие правки по возможным ошибкам OCR
- В оферте иногда используются сноски в виде звездочек - в этом случае просто вместо звездочки в тексте укажи содержимое сноски в скобочках.
- Между приложениями к договору делай разрыв через `---`
- Проверяй, что списки/параграфы не поехали
- В приложении с QR-кодом, если оно пустое, нужно докинуть редакторский текст, что QR-код нужно открыть в оригинальном документе
- При упоминании приложений в тексте документа, делай ссылки на эти приложения внутри документа
- При упоминании в тексте каких-то законов или норм - по возможности найди их в интернете и сделай из текста ссылку на них (например для статей ГК РФ, СанПин, распоряжений правительства и т.п.)
- При упоминании в тексте цветов RAL делай ссылку с них на https://ral.ru/ral8017 (номер подставь в урл)
- Исправь форматирование времени, если в оригинальном документе ошибка (в последней редакции видел, например, `9.00-18.00` -> `9:00-18:00`)
- После вложенных или маркированных списков отделяй следующий юридический пункт HTML-комментарием `<!-- break list -->`, а сам пункт пиши с начала строки: `1.2.`, `3.6.`, `4.1.1.`. Иначе Prettier вернет пункт внутрь предыдущего списка и сломает отступы в HTML.

- В каждом приложении первой строкой идет служебный текст, если это так - стоит его схлопнуть и обхединить заголовок так, чтобы он был таким "Приложение №X: [Тут текст заголовка ниже, который как раз описывает приложение]", пример ниже:
