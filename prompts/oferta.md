# Обновление оферт с сайта ОК Комфорт

Открой страницу https://ok-komfort.ru/objects/shelkovo-eco-club/ с помощью `agent-browser`. На ней опубликованы PDF-документы. Нас интересуют только публичные договоры.

Сначала составь список найденных договоров: название ссылки, исходный URL, slug части поселка и путь к KB-статье в `apps/www/src/data/kb/ok/offer`. Не обрабатывай приказы, регламенты и другие документы. Для каждого публичного договора запускай отдельного субагента. Обрабатывай договоры последовательно, не параллельно.

До окончания всей задачи веди журнал в `/private/tmp/opencode/oferta-progress.md`. Для каждого договора записывай исходный URL, slug, путь статьи, дату архива, S3 key и статусы скачивания, загрузки, OCR, редактирования и проверки. При повторном запуске продолжай с первого незавершенного этапа, не выполняя успешно завершенные этапы заново. После полного успеха удали журнал.

## Оригинал и архив

Для каждого договора:

1. Скачай исходный PDF без каких-либо преобразований во временную папку. Именно этот файл считается оригиналом и загружается в архив. OCR-версию в S3 не загружай.
2. Проверь через `file` и `pdfinfo`, что скачался корректный PDF, а не HTML-страница или ответ с ошибкой. Рассчитай SHA-256 файла.
3. Определи дату архива в формате `YYYY-MM-DD`:
   - если существующая KB-статья еще не содержит ссылки на архив `media.kpshelkovo.online`, это перенос старого материала: возьми filesystem creation date этой `.md`-статьи;
   - для новой статьи или новой редакции уже заархивированного договора используй фактическую дату скачивания;
   - дату редакции внутри договора для пути не используй.
4. Сформируй ключ `kb/ok/offer/<archive-date>/<slug>.pdf`, например `kb/ok/offer/2026-07-07/shelkovo-park.pdf`.
5. Перед загрузкой проверь, существует ли объект по этому ключу. Если он существует, скачай его и сравни SHA-256 с новым файлом. При совпадении не загружай повторно. При различии не перезаписывай исторический объект: останови обработку этого договора и явно сообщи о конфликте.
6. Загрузи оригинал в бакет `kps-public` с помощью `s5cmd`, endpoint `https://s3.regru.cloud` и явным MIME-типом `application/pdf`:

   ```bash
   s5cmd --endpoint-url "https://s3.regru.cloud" cp \
     --content-type "application/pdf" \
     --cache-control "public, max-age=31536000, immutable" \
     "<local-original.pdf>" \
     "s3://kps-public/kb/ok/offer/<archive-date>/<slug>.pdf"
   ```

7. Публичный URL объекта должен иметь вид `https://media.kpshelkovo.online/kb/ok/offer/<archive-date>/<slug>.pdf`. Проверь через CDN:
   - ответ `200`;
   - `Content-Type: application/pdf`;
   - размер равен размеру локального файла;
   - SHA-256 скачанного через CDN файла совпадает с локальным оригиналом.

Не считай загрузку завершенной, пока не прошли все четыре проверки.

## Распознавание и статья

После успешного архивирования оригинала:

1. Запусти OCR и сохрани результат в отдельный файл:

   ```bash
   ocrmypdf -l rus+eng --output-type pdfa --rotate-pages --deskew -O3 --jpeg-quality 85 \
     "<local-original.pdf>" "<local-ocr.pdf>"
   ```

2. Преобразуй OCR-версию в Markdown:

   ```bash
   docker run --rm -i adeuxy/markitdown:latest < "<local-ocr.pdf>" > "<parsed.md>"
   ```

3. Исправь в Markdown только явные опечатки и артефакты OCR. Ничего не придумывай и не дополняй от себя.
4. Создай или обнови статью `apps/www/src/data/kb/ok/offer/<slug>.md`.
5. Сразу после frontmatter добавь сноску с прямой ссылкой на архивный PDF:

   ```markdown
   > Документ распознан автоматически, перед использованием сверяйте текст с [оригиналом](https://media.kpshelkovo.online/kb/ok/offer/<archive-date>/<slug>.pdf).
   ```

6. После сноски добавь `[TOC]` отдельной строкой. При публикации новой редакции замени ссылку в сноске на новый архивный PDF.

## Индекс оферт

После обработки всех документов обнови индексный KB-документ `apps/www/src/data/kb/ok/index.md`: собери все оферты в одной группе и дай ссылки на статьи списком. Если для одной части поселка есть несколько оферт, сгруппируй их в одной строке.

## Правила редактирования

- Заголовок страницы: `Публичная оферта - [Название части поселка с нюансами из названия ссылки на исходной странице]`.
- Нельзя ничего придумывать или додумывать. Допустимы только небольшие исправления явных ошибок OCR.
- Если в оферте есть сноски со звездочками, замени звездочку в тексте содержимым сноски в скобках.
- Между приложениями к договору ставь `---`.
- Проверяй, что списки и параграфы сохранили структуру оригинала.
- Если приложение с QR-кодом после распознавания пустое, добавь редакторский текст о том, что QR-код нужно открыть в оригинальном PDF.
- Упоминания приложений превращай во внутренние ссылки на соответствующие разделы статьи.
- Упоминания законов и норм по возможности связывай с их официальными публикациями или надежными правовыми источниками. Не добавляй ссылку, если не уверен, что нашел нужный документ.
- Номера цветов RAL связывай с `https://ral.ru/ral<номер>`, например RAL 8017 — с `https://ral.ru/ral8017`.
- Исправляй очевидные ошибки в записи времени, например `9.00-18.00` на `9:00-18:00`.
- После вложенного или маркированного списка отделяй следующий юридический пункт HTML-комментарием `<!-- break list -->`, а сам пункт начинай с новой строки: `1.2.`, `3.6.`, `4.1.1.`. Иначе Prettier может вернуть пункт внутрь предыдущего списка и сломать HTML-разметку.
- Если приложение начинается со служебной строки и отдельного смыслового заголовка, объедини их в один заголовок.
