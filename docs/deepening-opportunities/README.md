# Идеи для углубления архитектуры

Эта папка хранит черновики архитектурных идей для будущего разбора. Это не ADR и не план внедрения. Каждая заметка описывает текущую боль, проверяет модуль через удаление и намечает направление улучшения без проектирования окончательного интерфейса.

Основание для заметок:

- `PRODUCT.md` - продуктовая рамка и доменные поверхности: новости, статус поселковых систем, тариф 815, сравнение поселков, люди и организации, поверхности для агентов.
- `apps/www/AGENTS.md` - локальные правила Astro-приложения, Markdown-пайплайн, URL/base для Compare и поверхности для агентов.
- `docs/decisions/*` - принятые ADR, которые не нужно заново спорить без реальной причины.
- `.agents/skills/improve-codebase-architecture/LANGUAGE.md` - архитектурный словарь. В заметках используются русские формулировки, но смысл совпадает с терминами про модуль, интерфейс, реализацию, глубину, границу, адаптер и близость к коду.

## Как читать

- `Проблема` описывает, где текущая архитектура заставляет держать знание в нескольких местах.
- `Проверка удалением` показывает, исчезает ли сложность при удалении модуля или расползается по вызывающему коду.
- `Идея углубления` описывает направление, но не задает финальный интерфейс.
- `Тестовый эффект` показывает, как должна сузиться тестовая поверхность: интерфейс становится главным предметом проверки.
- `Вопросы для обсуждения` нужны для следующего разговора перед любым изменением кода.
