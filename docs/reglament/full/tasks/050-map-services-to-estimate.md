# 050: Сопоставить услуги из полного регламента со сметой

## Цель

Понять, какие услуги из Приложения №4 явно есть в текущей смете калькулятора, какие есть частично, а какие не найдены отдельной строкой.

Это ключевая аудитная задача.

## Вход

- Контекст: `docs/reglament/full/000-agent-context.md`.
- Услуги: `docs/reglament/full/030-service-catalog.md`.
- Текущая смета: `apps/www/src/data/reglament/estimate-2026.ts`.
- Машиночитаемый feed, если удобнее: `/reglament/data/estimate-2026.json` после build/preview или напрямую через data module.

## Выход

Создать файл:

- `docs/reglament/full/050-service-to-estimate-map.md`.

## Статусы сопоставления

- `явно найдено` - есть понятная строка или секция сметы.
- `частично` - услуга похожа на группу строк, но не выделена прямо.
- `не найдено` - отдельной строки или понятного аналога не найдено.
- `требует проверки` - название, источник или детализация конфликтуют.

## Что проверить особенно

- Прием жителей в офисе.
- Газопровод после ввода в эксплуатацию.
- Очистка урн.
- Вывоз снега за пределы поселка.
- Ремонт объектов общего пользования.
- Ремонт ограждения против строки ремонта дорог/площадок.

## Acceptance Criteria

- [ ] Есть `docs/reglament/full/050-service-to-estimate-map.md`.
- [ ] Каждая услуга из `030-service-catalog.md` имеет статус сопоставления.
- [ ] Для статусов `не найдено` и `требует проверки` есть короткое пояснение.
- [ ] Нет изменений в `apps/www`.

## Зависимости

- `030-extract-service-catalog.md`.
- Текущая нормализованная смета в `estimate-2026.ts`.
