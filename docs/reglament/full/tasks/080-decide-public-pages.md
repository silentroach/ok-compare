# 080: Решить, какие публичные страницы делать

## Цель

На основе извлеченных данных решить, какие отдельные страницы действительно нужны людям, а какие данные оставить только для LLM.

## Вход

- Контекст: `docs/reglament/full/000-agent-context.md`.
- Все markdown-артефакты `020-070`.

## Выход

Создать файл:

- `docs/reglament/full/080-public-pages-decision.md`.

## Кандидаты страниц

- `/reglament/assets/` - общее имущество.
- `/reglament/services/` - услуги и периодичность.
- `/reglament/villages/` - характеристики поселков.

## Что решить

- Какие страницы делать в первом релизе.
- Какие оставить только как LLM dataset.
- Какие ссылки добавить с `/reglament/`.
- Какие страницы нуждаются в таблицах, а какие только в кратком тексте.

## Acceptance Criteria

- [ ] Есть `docs/reglament/full/080-public-pages-decision.md`.
- [ ] Для каждой кандидатной страницы есть решение: `делать`, `позже`, `не делать`.
- [ ] Есть короткое объяснение, почему `/reglament/` остается калькулятором.
- [ ] Нет изменений в `apps/www`.
