# 010: Составить карту страниц полного регламента

## Цель

Понять структуру `docs/reglament/original/full.pdf` и отметить, какие страницы нужны для следующих задач.

Не извлекать все данные подробно. Эта задача только про навигационную карту PDF.

## Вход

- Контекст: `docs/reglament/full/000-agent-context.md`.
- Источник: `docs/reglament/original/full.pdf`.

## Выход

Создать файл:

- `docs/reglament/full/010-page-map.md`.

## Что сделать

- Выписать диапазоны страниц и краткое содержание.
- Разметить каждый диапазон статусом: `брать`, `пропустить`, `позже`, `уже покрыто сметой`.
- Отдельно отметить страницы для следующих задач:
  - характеристики поселков;
  - состав общего имущества;
  - перечень услуг и периодичность;
  - расчетные допущения, если они потенциально спорные.
- Не разбирать методические основания и карты, только отметить их как `пропустить` или `позже`.

## Acceptance Criteria

- [ ] Есть `docs/reglament/full/010-page-map.md`.
- [ ] В файле перечислены все крупные разделы PDF с диапазонами страниц.
- [ ] Страницы приложений 1, 2 и 4 явно отмечены как приоритетные.
- [ ] Методические основания и карты не уходят в подробную обработку.
- [ ] Нет изменений в `apps/www`.

## Зависимости

Нет.
