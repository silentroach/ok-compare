Рабочие входы этой задачи: извлеченные артефакты `020` - `070`.

PDF: `full`.

## Что идет только в LLM dataset в первом релизе

| Данные                             | Где хранить                                                        | Публичное использование                                                                                                                                                                 |
| ---------------------------------- | ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `villages[]`                       | JSON/Markdown dataset по contract из `070-llm-dataset-contract.md` | Использовать как короткую справку в калькуляторе или на страницах `assets/services`, но не выводить отдельной страницей в первом релизе.                                                |
| `calculation_assumptions[]`        | Dataset                                                            | Не делать отдельную страницу с расчетными допущениями. Выводить точечные осторожные пояснения там, где они нужны: единый тариф, расчет по соткам, `м²` против соток.                    |
| `audit_notes[]`                    | Dataset                                                            | Не делать отдельную публичную страницу аудита. Использовать как предупреждения `требует проверки`, `не найдено в явной строке сметы`, `частично сопоставлено` на релевантных страницах. |
| Полное `service_to_estimate_map[]` | Dataset и страница `/reglament/services/` в сокращенном виде       | На публичной странице показывать человекочитаемые статусы и объяснения, а полный машинный слой оставить для LLM/API.                                                                    |

## Почему `/reglament/` остается калькулятором

`/reglament/` уже отвечает на главный пользовательский вопрос: сколько стоит тариф по площади участка. Если перенести на главную таблицы имущества, перечень услуг и проверочные заметки, калькулятор станет справочником и потеряет основной сценарий.

Главная должна остаться точкой расчета тарифа. Новый слой полного регламента лучше подключать коротким блоком ссылок: кто хочет проверить состав имущества или перечень услуг, уходит на отдельные страницы; кто хочет посчитать платеж, остается на калькуляторе.

В первом релизе добавить на главную только компактный блок после основного результата или рядом с пояснениями к расчету:

| Текст ссылки                        | URL                    | Назначение                                                                 |
| ----------------------------------- | ---------------------- | -------------------------------------------------------------------------- |
| `Что входит в общее имущество`      | `/reglament/assets/`   | Перевести пользователя к таблицам имущества из Приложения №2.              |

## Требования к страницам первого релиза

### `/reglament/assets/`

- Нужны таблицы: данные многомерные, есть категории, поселки, единицы измерения, итоги и заметки по проверке.
- Группировать по категориям из `020-common-assets.md`: дороги и тротуары, зеленые насаждения, объекты благоустройства, электроснабжение, охрана.
- Показывать пустые ячейки как `-` или `нет значения в таблице PDF`, но не как подтвержденный ноль.
- Для строк с пометками из `020` сохранять осторожные формулировки: `требует визуальной сверки`, `пустые ячейки в PDF`, `единица измерения не указана`.
- Источник страницы: `full`, Приложение №2, pages 128-130.

### `/reglament/services/`

- Нужны таблицы: минимум таблица услуг по группам `Круглогодично`, `В зимний период`, `В летний период`; отдельно краткая таблица сопоставления со сметой.
- Показывать 24 услуги из `030-service-catalog.md` с периодичностью и page 135.
- Статусы сопоставления брать из `050-service-to-estimate-map.md`; нормализация для данных описана в `070` как `explicit_found`, `partial`, `not_found`, `needs_check`.
- Не превращать статусы `не найдено` в обвинение или юридический вывод; использовать формулировки `не найдено в явной строке сметы`, `частично сопоставлено`, `требует проверки`.
- Источник услуг: `full`, Приложение №4, page 135; источник сверочных строк сметы: `full`, pages 125-126.

### `/reglament/villages/`

- В первом релизе не делать.
- Если делать позже, страница должна быть короткой: одна таблица с 4 поселками, домовладениями, площадью в сотках и расчетной долей площади.
- Обязательно объяснить, что доли площади рассчитаны из page 127, а не напечатаны отдельной строкой PDF.
- Не добавлять карты поселков и расшифровку отдельных участков: этих данных нет в извлеченных артефактах `020-070`.

## Проверка решения

- Для каждой кандидатной страницы есть решение: `делать`, `позже`, `не делать`.
- Первый релиз содержит только `/reglament/assets/` и `/reglament/services/`.
- `/reglament/` остается калькулятором и получает только короткие ссылки на новые справочные страницы.
- Данные `villages[]`, `calculation_assumptions[]`, `audit_notes[]` остаются частью LLM dataset и не требуют отдельных страниц в первом релизе.
- Изменений в `apps/www` в этой задаче нет.
