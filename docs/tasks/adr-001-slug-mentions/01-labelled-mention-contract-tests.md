# Задача 1: Зафиксировать контракт упоминаний с авторским текстом

## Описание

Добавить тесты, которые описывают новый синтаксис `[видимый текст](@slug)` для людей до изменения реализации. Задача должна явно показать, что авторский текст сохраняется, адрес превращается в URL профиля, обычные ссылки не ломаются, а `[текст](@slug:case)` не является поддерживаемым синтаксисом.

## Критерии приемки

- [x] `normalizePeopleMentions` возвращает Markdown с замененным адресом ссылки: `[главного по электричеству](/people/kschemelinin/)`.
- [x] `normalizePeopleMentions` добавляет человека из `[текст](@slug)` в `mentions` так же, как из `@slug`.
- [x] `renderMarkdown` рендерит `[текст](@slug)` в HTML-ссылку на профиль и сохраняет видимый текст.
- [x] Обычная ссылка вроде `[канал](https://example.com)` остается без изменений.
- [x] `[текст](@slug:gen)` покрыт тестом как ошибка или явно неподдерживаемый формат.
