# Task 10: Почистить старые вспомогательные функции и проверить миграцию

## Статус

Выполнено

Заметка: старый app-helper `apps/www/src/lib/markdown/frontmatter.ts` удален как неиспользуемый. Для оставшихся line-based публичных Markdown/llms документов добавлен общий app-helper поверх `@shelkovo/markdown`, который выделяет заголовки AST-узлами и парсит только секционные Markdown-блоки, а не сериализует целый документ локальным `lines.join('\n')`.

## Навыки

- `test-driven-development`
- `code-review-and-quality`

## Цель

Завершить миграцию: убрать старые ручные вспомогательные функции, обновить документацию пакета и проверить, что публичный Markdown больше не генерируется строковым способом.

## Контекст

После задач 01-09 все сопровождающие Markdown-страницы и llms должны использовать пакетный AST API. Последняя задача нужна для уборки хвостов и общей проверки. Ограничение импортов по ADR-008 не нужно.

## Что сделать

- Проверить, остался ли `apps/www/src/lib/markdown/frontmatter.ts` нужным. Если нет, удалить его.
- Проверить импорты старой вспомогательной функции frontmatter и удалить оставшиеся только после миграции всех потребителей.
- Обновить `packages/markdown/README.md`, если предыдущие задачи не полностью описали API генерации.
- Проверить `apps/www/AGENTS.md` и корневой `AGENTS.md`: если правила про генерацию Markdown стали неполными, обновить кратко и без дублирования ADR.
- Найти оставшиеся публичные Markdown/llms генераторы, которые продолжают собирать целый документ через ручные массивы строк, и либо мигрировать, либо явно задокументировать причину исключения.
- Выполнить финальные проверки.

## Чего не делать

- Не добавлять ограничение импортов, lint-правило или архитектурный тест против ручной генерации.
- Не переписывать обычные строковые вспомогательные функции форматирования, которые не сериализуют Markdown-документ.
- Не менять публичный текст ради редакционной правки.
- Не трогать несвязанные untracked или пользовательские файлы.

## Критерии приемки

- Старая вспомогательная функция frontmatter в приложении удалена или имеет явно задокументированную причину существования.
- Документация `@shelkovo/markdown` объясняет разницу между рендером Markdown в HTML и генерацией Markdown через AST.
- В публичных Markdown/llms генераторах нет ручной сериализации целого документа через `lines.join('\n')` или аналогичную локальную вспомогательную функцию.
- ADR-008 и task README не противоречат фактической реализации.

## Проверка

- Запустить `pnpm --filter @shelkovo/markdown test`.
- Запустить `pnpm --filter @shelkovo/markdown typecheck`.
- Запустить `pnpm --filter @shelkovo/www test`.
- Запустить `pnpm --filter @shelkovo/www typecheck`.
- Запустить `pnpm build` или явно записать причину, если build не может быть выполнен в текущей среде.
