# Page Meta Style

## Title

- `title` видит пользователь, поэтому он должен читаться как чистая вкладка браузера, а не как набор SEO-ключей.
- Единственный разделитель уровней: длинное тире `—`.
- Бренд или продукт ставится в конец: `Страница — Раздел — Бренд`.
- Главная страница остается только брендом, если нет конкретной причины добавить контекст.
- Для корневого сайта использовать `Шелково Онлайн`, для compare-продукта: `Сравни с Шелково`.
- Для обычных пользователей выбирать понятные слова вместо внутренних ярлыков: `Темы новостей`, а не `Теги`.
- Внутри названия страницы можно использовать двоеточие: `Поселок: тариф и инфраструктура — Сравни с Шелково`.
- Не добавлять в title домены, юрлица, повторы `Шелково`, технические слова и рекламную воду.
- Держать `title` около 60 символов; для динамических detail-страниц считать 70 символов верхней границей и сокращать шаблон или данные.
- В контентных полях вроде `seo.title` хранить только уникальную часть title; секцию и бренд достраивать в шаблоне страницы. Добавлять такие поля только когда обычный заголовок слишком длинный.

Хорошие примеры:

- `Шелково Онлайн`
- `Новости — Шелково Онлайн`
- `Вода — Статус — Шелково Онлайн`
- `Сравнение поселков — Сравни с Шелково`
- `Шелково Ривер: тариф и инфраструктура — Сравни с Шелково`

## Description

- `description` не показывается как заголовок вкладки, поэтому оптимизировать его под превью и поисковый сниппет.
- Писать одну или две короткие фразы о содержании страницы.
- Называть полезный охват: новости, период архива, статус сервиса, тариф поселка, инфраструктуру или контекст источников.
- Не использовать внутренние слова вроде `body`, `addenda`, `pinned`, `reshuffle`, названия маршрутов и детали реализации.
- Не набивать description ключевиками и повторяющимися брендовыми фразами.
- Для динамических страниц собирать description из реальных данных страницы, если это улучшает сниппет.
- Держать `description` в пределах 160-170 символов. Если пользовательский текст нужен в карточках, RSS или feeds, не сокращать его ради meta: добавить отдельный `seo.description`.
