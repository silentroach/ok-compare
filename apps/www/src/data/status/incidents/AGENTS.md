# AGENTS.md

Локальные правила для `src/data/status/incidents`.

- По умолчанию делай минимальную запись: только frontmatter.
- Для записей без body предпочитай короткий slug файла: `service-outage-YYYY-MM-DD.md` для всего поселка и `service-area-outage-YYYY-MM-DD.md` для локальной зоны.
- Если в один день есть несколько однотипных записей с одинаковым scope, добавляй к slug стартовое время в формате `-HHMM`.
- Не кодируй в slug лишние слова вроде `settlement-wide`, `morning`, `day`, `night`, если это уже следует из frontmatter.
- Не добавляй `title`, если хватает автогенерируемого заголовка по `service` + `kind`.
- Не добавляй body, если он только повторяет факт инцидента, время и охват из frontmatter.
- Добавляй `title` и/или body только если есть дополнительная ценность: причина, последствия, ход восстановления, уточнение по зонам, важная цитата или иной контекст из источника.
- Не выдумывай описательный текст ради наличия body.
- Запись без body остается в feed и таймлайнах, но не получает detail page.
