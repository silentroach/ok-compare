# AGENTS.md

Локальные правила для `src/data/people`.

- Один человек = один markdown-файл без вложенных директорий.
- Имя файла без расширения должно быть canonical slug в lower-case Latin с дефисами.
- `name` обязателен и должен быть каноническим публичным именем человека в именительном падеже.
- Если в материале нужен другой падеж mention, добавь форму в `name_cases` этого человека (`gen`, `dat`, `acc`, `ins`, `prep`) и только потом используй `@slug:case`.
- `contacts` обязателен как массив объектов `{ type, value }`, даже если он пустой по редакционному решению.
- В MVP `contacts[].type` ограничен значениями `phone` и `telegram`.
- `contacts[].value` не должен быть пустым или состоять только из пробелов.
- Body опционален. Если он есть, это свободный редакционный markdown-текст, который добавляет контекст помимо будущих backlinks.
- Если человек упоминается в `news`, `status` или другом person profile, в source нужно писать `@slug`, `@slug:case` или `[видимый текст](@slug)`, а не plain text имя и не ручную ссылку на `/people/.../`.
- `@slug` использовать для канонического имени в именительном падеже; `@slug:case` - для имени в другом падеже после добавления формы в `name_cases`.
- `[видимый текст](@slug)` использовать для роли, описания или фразы, которая должна остаться авторским видимым текстом.
- `[текст](@slug:case)` не использовать: в labelled mention нужная грамматика должна быть написана в видимом тексте.
- Неизвестные mention-ключи и запрошенные, но незаполненные падежи не допускаются: source/profile нужно исправлять до merge.
