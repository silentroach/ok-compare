# AGENTS.md

Локальные правила для `src/data/meetings`.

- Одна встреча = одна slug-директория в lower-case Latin с цифрами и дефисами.
- В каждой директории встречи обязательны `index.yaml` и `transcript.yaml`.
- Не создавай встречу без полной транскрипции.
- Если одна встреча состоит из нескольких записей, добавляй `transcript-2.yaml`, `transcript-3.yaml` и так далее; внутри каждого файла время начинается заново с `00:00:00`.
- Спикеров описывай в `index.yaml`; transcript-файлы хранят только `segments`.
- В `speakers` используй `person` только для существующих `/people/` профилей.
- Для разовых спикеров используй локальное поле `name`.
- `segments[].text` - plain text, не Markdown; сохраняй речь, не превращай в протокол.
- Не добавляй в meeting data протокол, решения, action items или редакционные выводы.
- Из новостей ссылайся обычными Markdown-ссылками на `/meetings/[slug]/#t-HH-MM-SS`; для повторяющихся времен во втором и следующих файлах используй сгенерированный суффикс якоря.
