# explorer-data

## Когда использовать

- Когда нужен один структурированный источник со всеми подтвержденными данными по поселкам.
- Когда нужно сравнить рейтинг, тариф, инфраструктуру, сервисы и расстояния без загрузки детальных страниц.
- Когда нужен слой обнаружения перед переходом к конкретному поселку.

## Что открыть

- Основной URL: `/data/settlements.json`.
- Смотри прежде всего на `settlements[]`, `comparisons` и `stats`.
- Для минимального набора данных и повторения логики главной есть `/data/explorer.json`.
- Для перехода к детальной странице используй `settlements[].slug`.

## Поля, полезные для массового анализа

- `short_name`, `slug`, `website`, `telegram`, `management_company`, `rating`, `is_baseline`
- полный `location` и полный `tariff`
- `infrastructure`, `common_spaces`, `service_model`, `distance.moscow_km`, `distance.mkad_km`, `distance.shelkovo_km`
- `stats` для общих тарифных агрегатов и медианы похожих поселков внутри рейтинговой группы Шелково
- `comparisons[slug]` для дельты тарифа относительно базового поселка Шелково

## Ограничения

- Это полная структурированная лента, но не замена HTML- или Markdown-странице как человекочитаемому представлению.
- `data/explorer.json` остается отдельной минимальной лентой для списка и карты.
- Список `sources` не входит в общую ленту и остается на детальной странице поселка.
- Отсутствие поля обычно означает «неизвестно», а не «точно нет».

## Дальше

- Если нужен URL страницы или человекочитаемый контекст по одному поселку, переходи на `/settlements/[slug]/`.
