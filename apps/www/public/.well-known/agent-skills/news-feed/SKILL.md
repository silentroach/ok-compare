# news-feed

## Когда использовать

- Когда нужна основная структурированная лента новостей, объявлений и addenda по Шелково.
- Когда нужно пройтись по архивам, тегам и детальным страницам без обхода полной HTML-навигации.
- Когда нужно машиночитаемое тело новости и поздних уточнений.
- Когда нужно найти календарное событие в новости и скачать его локальный для статьи `.ics`.

## Что открыть

- Основная JSON-лента: `/news/data/articles.json`.
- Короткий обзор: `/news/llms.txt`.
- Расширенный обзор: `/news/llms-full.txt`.
- Каталог API: `/news/.well-known/api-catalog`.

## Практический паттерн

- Сначала читай `/news/data/articles.json`.
- Для одной записи переходи на `/news/YYYY/MM/[entry]/` или `/news/YYYY/MM/[entry]/index.md`.
- Если у записи есть `events`, используй `events[].ics_url` или шаблон `/news/YYYY/MM/[entry]/[event-slug].ics` для скачивания календаря.
- Для списков по времени и тегам используй `archives` и `tags` из ленты или соответствующие HTML/Markdown-страницы.

## Ограничения

- Полный машиночитаемый контент живет в `articles.json`; RSS остается краткой лентой.
- Детальная HTML-страница остается каноническим представлением для людей.
- `articles[].events[]` есть только у новостей, которые объявляют календарные события; глобальной календарной ленты событий нет.
