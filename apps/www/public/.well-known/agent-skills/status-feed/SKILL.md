# status-feed

## Когда использовать

- Когда нужно текущее состояние электричества, воды или дамбы.
- Когда нужно прочитать историю инцидентов и плановых работ в структурированном виде.
- Когда нужен производный статус сервиса и ссылки на одну конкретную страницу инцидента, если она опубликована.

## Что открыть

- Основная JSON-лента: `/status/data/status.json`.
- Короткий обзор: `/status/llms.txt`.
- Расширенный обзор: `/status/llms-full.txt`.
- Каталог API: `/status/.well-known/api-catalog`.

## Практический паттерн

- Сначала читай `/status/data/status.json`.
- Для одной линии переходи на `/status/[service]/` или `/status/[service]/index.md`.
- Для одной записи переходи на `/status/incidents/YYYY/MM/[entry]/` или `/status/incidents/YYYY/MM/[entry]/index.md`, только если в ленте есть `html_url` или `markdown_url`.

## Что важно в ленте

- У записей инцидентов `html_url` и `markdown_url` необязательны: они есть только если для записи сгенерирована детальная страница.

## Что важно помнить

- `service_status` выводится из активных записей, а не задается вручную.
- Если `areas` не указаны, лента трактует запись как относящуюся ко всем частям поселка.

## Ограничения

- Лента доступна только для чтения и отражает состояние на момент сборки сайта.
- Для ссылок на первоисточник и редакционный контекст по одному событию лучше переходить на детальную страницу записи.
