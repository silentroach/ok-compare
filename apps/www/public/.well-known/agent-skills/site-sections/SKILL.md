# site-sections

## Когда использовать

- Когда агент начинает с корня `kpshelkovo.online` и еще не знает, какой раздел ему нужен.
- Когда нужно быстро найти правильную точку входа для новостей, статуса, профилей людей или сравнения тарифов поселков.
- Когда нужен обзор разделов и ссылок для обнаружения, а не детальное чтение одной конкретной ленты.

## Что открыть

- Корневой `llms.txt`: `/llms.txt`.
- Расширенный обзор: `/llms-full.txt`.
- Markdown-версия главной: `/index.md`.
- Каталог API сайта: `/.well-known/api-catalog`.

## Как выбирать раздел

- `/news/` — новости, объявления, addenda, необязательные события и архивы; основная лента: `/news/data/articles.json`, календарь события: `/news/YYYY/MM/[entry]/[event-slug].ics`.
- `/status/` — текущее состояние сервисов, инциденты и плановые работы; основная лента: `/status/data/status.json`.
- `/people/index.md` — Markdown-обзор профилей людей без публичного HTML-индекса; основная лента: `/people/data/people.json`.
- `/815/compare/` — сравнение тарифов поселков и рейтинга; основная лента: `/815/compare/data/settlements.json`.

## Ограничения

- JSON-ленты доступны только для чтения и отражают состояние на момент сборки сайта.
- Публичные навыки на корне покрывают навигацию, новости, статус и профили людей. У Compare есть свой отдельный индекс навыков.
